# simpleHPC 产品增强计划：License 监控与项目工作流管理

版本：v0.2.x 规划草案
日期：2026-07-05
状态：待评审

## 1. 背景

当前 simpleHPC 已具备集群基础管理能力，包括用户、角色、队列、作业、数据目录、VNC、WebSSH、巡检、日志、RBAC 等模块。后续如果面向企业、高校、科研院所和智算中心真实生产场景，需要继续补齐两个关键业务能力：

1. 商业软件 License 使用量监控；
2. 面向项目 / 课题 / 研发任务的项目协作与工作流管理。

这两个方向分别解决：

- License 监控：商业软件授权是否够用、谁在用、哪些应用即将耗尽授权；
- 项目工作流：用户作业不再是孤立任务，而是围绕某个项目目标形成完整计算过程。

## 2. 产品主线重新划分

| 主线 | 面向对象 | 主要目标 | 典型模块 |
|---|---|---|---|
| 平台管理线 | 集群管理员、配置管理员、运维人员 | 保证集群稳定、安全、高效运行 | 集群配置、节点监控、队列管理、QOS、巡检、告警、日志、RBAC、License |
| 用户工作台线 | 普通用户、组长、单位管理员、项目成员 | 提交作业、使用数据、使用终端和桌面 | 作业模板、作业列表、数据目录、WebSSH、VNC |
| 项目协作线 | 项目负责人、项目成员、跨团队协作者 | 围绕项目组织数据、任务、作业、结果和报告 | 项目中心、项目任务、项目数据、项目作业、项目工作流、项目报告 |

## 3. 方向一：商业软件 License 使用量监控

### 3.1 建设目标

为 HPC 集群中的商业软件提供 License 使用量可视化、趋势分析、告警和审计能力。

典型软件包括：

- ANSYS
- Abaqus
- COMSOL
- MATLAB
- STAR-CCM+
- Fluent
- Gaussian
- Materials Studio
- 其他 FlexLM / RLM / Sentinel / 自定义 License 服务

### 3.2 核心功能

| 功能模块 | 功能说明 | 优先级 |
|---|---|---|
| License 服务配置 | 配置 License 服务地址、端口、类型、采集命令 | P0 |
| License 总览 | 展示总点数、已用点数、可用点数、使用率 | P0 |
| 应用 Feature 监控 | 按软件 / Feature 展示授权使用情况 | P0 |
| 使用用户明细 | 查看当前占用 License 的用户、主机、开始时间 | P1 |
| 历史趋势 | 展示 License 使用率历史趋势 | P1 |
| 告警规则 | 使用率超过阈值、License 服务不可用时告警 | P1 |
| 飞书通知 | License 紧张或服务异常时发送通知 | P1 |
| 报表导出 | 按月导出 License 使用报告 | P2 |
| License 成本核算 | 结合项目 / 用户 / 单位统计商业软件使用成本 | P2 |

### 3.3 页面规划

建议新增一级菜单：

```text
运维管理
  - License 监控
```

License 监控页面建议包含：

1. License 服务状态卡片；
2. 软件授权使用率排行；
3. 当前占用明细；
4. 历史趋势图；
5. 告警规则配置；
6. 服务配置入口。

### 3.4 数据采集方式

第一阶段优先支持 FlexLM：

```bash
lmstat -a -c port@host
lmutil lmstat -a -c port@host
```

采集方式：

- 后端定时任务每 1-5 分钟采集；
- 解析输出为结构化数据；
- 写入采样表；
- 前端读取聚合结果。

| `license_features` | 软件 Feature 配置 |

| `license_alert_rules` | 告警规则 |

### 3.7 权限设计

| 角色 | 权限 |
|---|---|
| cluster_admin | 查看和管理全部 License 配置与数据 |
| config_admin | 可配置 License 服务，可查看全局聚合数据 |

## 4. 方向二：项目 / 课题 / 工作流管理

### 4.1 建设目标

将当前“孤立作业”升级为“围绕项目组织的数据、任务、作业、结果和报告”。

项目可以对应：

- 企业研发项目；
- 高校课题；
- 仿真计算任务包；
- 科研分析流程；
- AI4S 计算流程；
- 跨团队协作项目。

### 4.2 关键判断

项目权限不是简单等同于用户组权限。

当前权限主要按单位、团队、个人划分：

```text
单位 -> 团队 / 用户组 -> 用户
```

但项目可能跨单位、跨团队、跨用户组。因此需要新增一套项目维度权限：

```text
平台 RBAC 权限
+ 文件目录边界权限
+ 项目空间权限
```

项目权限只在项目范围内生效，不应该破坏现有普通用户目录边界和团队目录边界。

### 4.3 项目权限示例

```text
user001 属于 A 团队
user002 属于 B 团队

默认情况下：
user001 不能访问 B 团队数据
user002 不能访问 A 团队数据

如果 user001 和 user002 都加入 Project-X：
两人都可以访问 Project-X 的项目数据空间
两人都可以查看 Project-X 中授权范围内的作业和结果
但仍不能访问彼此团队的非项目数据
```

### 4.4 建议新增模块

新增一级菜单：

```text
项目中心
```

二级菜单：

```text
项目总览
我的项目
项目任务
项目数据
项目作业
项目工作流
项目报告
项目资源统计
```

### 4.5 核心功能

| 功能模块 | 功能说明 | 优先级 |
|---|---|---|
| 项目管理 | 创建项目、维护项目基本信息、周期、状态 | P0 |
| 项目成员 | 添加跨单位、跨团队成员，配置项目内角色 | P0 |
| 项目数据空间 | 为项目创建独立数据目录，按项目权限访问 | P0 |
| 项目作业绑定 | 提交作业时选择所属项目，项目内可查看关联作业 | P0 |
| 项目任务 | 拆分项目任务，设置负责人、状态、截止时间 | P1 |
| 任务依赖 | 建立任务之间的前后依赖关系 | P1 |
| 工作流视图 | 以流程图方式展示任务、作业、数据关系 | P2 |
| 项目报告 | 项目结束时生成项目计算报告和归档 | P2 |
| 项目资源统计 | 统计项目 CPU/GPU/存储/License 使用量 | P2 |

### 4.6 项目内角色建议

| 项目角色 | 权限说明 |
|---|---|
| 项目负责人 | 管理项目、成员、任务、数据和作业 |
| 项目管理员 | 协助管理任务、数据和作业 |
| 计算成员 | 提交和管理自己的项目作业 |
| 数据成员 | 管理项目数据空间 |
| 只读成员 | 查看项目进展、结果和报告 |
| 外部协作者 | 受限访问指定任务或目录 |

### 4.7 项目数据空间设计

建议项目目录统一放在：

```text
/data/projects/{projectCode}
```

或：

```text
/data/project/{projectId}
```

项目数据空间权限：

| 权限 | 说明 |
|---|---|
| read | 读取项目文件 |
| upload | 上传文件 |
| download | 下载文件 |
| write | 新建、修改文件 |
| delete | 删除项目文件 |
| manage | 管理项目目录和授权 |

注意：

- 项目数据空间必须由后端做路径边界校验；
- 不能让项目权限扩大到用户个人目录或团队非项目目录；
- 项目目录可以通过软链接或挂载方式和存储系统集成，但后端必须校验真实路径。

### 4.8 项目作业设计

作业提交时增加字段：

```text
project_id
project_task_id
```

项目详情中可以查看：

- 项目下所有作业；
- 作业提交人；
- 作业模板；
- 作业状态；
- 标准输出 / 标准错误；
- 结果文件；
- 资源消耗；
- 失败原因；
- 重跑记录。

### 4.9 项目任务和工作流设计

第一阶段不建议直接做复杂 DAG 编排，可以先做轻量任务依赖。

任务字段：

- 任务名称；
- 任务负责人；
- 所属项目；
- 任务状态；
- 上游依赖任务；
- 关联作业；
- 关联输入数据；
- 关联输出结果；
- 截止时间；
- 任务说明。

典型流程：

```text
数据准备 -> 数据清洗 -> 网格生成 -> 仿真计算 -> 结果分析 -> 报告输出
```

## 5. 总体实施计划

### 5.1 阶段划分

| 阶段 | 名称 | 目标 | 预计工作量 | 优先级 |
|---|---|---|---|---|
| 阶段 1 | License 监控 MVP | 支持 FlexLM 服务配置、采集、总览、当前占用 | 中 | P0 |
| 阶段 2 | License 告警与趋势 | 支持历史趋势、阈值告警、飞书通知 | 中 | P1 |
| 阶段 3 | 项目中心 MVP | 支持项目、成员、项目数据空间、项目作业绑定 | 大 | P0 |
| 阶段 4 | 项目任务管理 | 支持任务拆分、负责人、状态、依赖和关联作业 | 中 | P1 |
| 阶段 5 | 项目工作流和报告 | 支持流程视图、项目报告、归档和资源统计 | 大 | P2 |
| 阶段 6 | 项目 + License + 成本分析 | 按项目统计 CPU/GPU/存储/License 消耗 | 大 | P2 |

### 5.2 推荐先后顺序

建议先做 License 监控 MVP，再做项目中心 MVP。

原因：

1. License 监控相对独立，对现有权限和文件系统影响较小；
2. 项目中心会影响用户、团队、作业、文件、RBAC，必须先完成详细设计；
3. License 使用量后续可以自然接入项目资源统计。

## 6. 第一阶段详细计划：License 监控 MVP

| 任务 | 内容 | 交付物 |
|---|---|---|
| 1 | 增加 License 服务配置表 | DB migration |
| 2 | 增加 FlexLM 采集脚本和解析器 | 后端 collector |
| 3 | 增加 License 采样表 | DB migration |
| 4 | 增加 License API | `/api/v1/licenses/*` |
| 5 | 增加 License 监控页面 | 前端页面 |
| 6 | 增加 RBAC 权限点 | 菜单、路由、接口权限 |
| 7 | 增加采集失败日志和审计 | 系统日志 / 审计日志 |
| 8 | 测试服务器部署验证 | 验收记录 |

验收标准：

- 可以配置一个 FlexLM License 服务；
- 可以手动触发采集；
- 可以展示总点数、已用点数、可用点数；
- 可以展示当前使用用户和主机；
- License 服务不可用时有明确错误提示；
- 普通用户不能看到其他用户明细；
- RBAC enforce 下权限正常。

## 7. 第二阶段详细计划：项目中心 MVP

| 任务 | 内容 | 交付物 |
|---|---|---|
| 1 | 编写项目中心详细设计 | `docs/project-center-design.md` |
| 2 | 增加项目相关数据表 | DB migration |
| 3 | 增加项目权限内核 | 项目成员 / 项目角色 / 项目数据范围 |
| 4 | 增加项目 API | `/api/v1/projects/*` |
| 5 | 增加项目中心页面 | 项目列表、详情、成员 |
| 6 | 增加项目数据空间 | 项目文件列表、上传、下载 |
| 7 | 作业提交绑定项目 | 模板提交 / 普通提交增加项目选择 |
| 8 | 项目详情展示作业 | 项目作业列表 |
| 9 | RBAC + 项目权限联动测试 | 权限测试 |
| 10 | 测试服务器部署验证 | 验收记录 |

验收标准：

- 可以创建项目；
- 可以添加跨团队成员；
- 项目成员可以访问项目数据空间；
- 非项目成员不能访问项目数据；
- 作业可以绑定项目；
- 项目详情可以查看项目作业；
- 普通用户不能因为项目权限突破个人目录和团队非项目目录；
- RBAC enforce 下权限正常。

## 8. 风险与注意事项

| 风险 | 说明 | 处理建议 |
|---|---|---|
| 项目权限和现有 RBAC 冲突 | 项目权限是业务权限，不能覆盖平台安全边界 | 设计项目权限叠加模型 |
| 文件路径越权 | 项目目录可能引入新的跨团队访问路径 | 后端统一做路径规范化和边界校验 |
| 作业归属复杂 | 作业既属于用户，也可能属于项目和任务 | 作业表增加项目关联，不删除用户归属 |
| License 解析差异 | 不同软件 License 输出格式不同 | 第一阶段只支持 FlexLM，后续插件化 |
| 告警噪声 | License 短时波动可能频繁告警 | 增加持续时间和抑制窗口 |
| 项目功能过重 | 一次性做完整工作流容易失控 | 先做项目、成员、数据、作业绑定 MVP |

## 9. 建议结论

建议将 simpleHPC 后续产品定位升级为：

```text
面向 HPC / AI4S / 工程仿真 / 科研计算的集群管理与项目协作平台
```

短期优先级：

1. 先做 License 监控 MVP；
2. 同步编写项目中心详细设计；
3. 再做项目中心 MVP；
4. 最后把项目、作业、数据、License、资源消耗串成完整生产闭环。
