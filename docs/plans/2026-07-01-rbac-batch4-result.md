# RBAC 批次四交付记录：前端动态权限和角色管理

## 1. 执行边界

- 权限模式保持 `legacy/shadow`，未切换 `enforce`。
- 未部署生产环境或测试服务器业务目录。
- 未修改现有业务数据库。
- 新增迁移仅以 up/down 文件交付，等待批次五在隔离测试库执行。

## 2. 前端权限客户端

`js/rbac.js` 以 `/api/v1/auth/me` 为唯一运行时权限事实来源，提供：

- 权限上下文加载和主动刷新；
- `can(permission)` 权限判断；
- 动态菜单渲染；
- 页面路由守卫；
- `[data-permission]` 按钮显示/禁用控制；
- `simplehpc:authz-ready` 和 `simplehpc:forbidden` 页面事件。

页面在权限上下文完成前进入 `rbac-pending` 状态，避免未授权页面内容短暂闪现。

## 3. 动态菜单和路由

- 普通用户使用后端 `flatMenu=true` 结果，展示 6 个一级菜单：
  仪表盘、队列状态、数据目录、作业模板、作业列表、VNC 桌面。
- 普通用户菜单无分组标题。
- 管理员和自定义角色按后端菜单树渲染。
- 页面访问同时检查后端返回菜单路径和 `routePermission`。
- 无权访问时显示 403，并提供第一个可访问页面入口。
- `MenuItem` 新增 `routePermission` 和 `resource` 元数据，供路由守卫和权限矩阵使用。

## 4. 角色管理

角色工作台支持：

- 角色列表、新建、编辑、复制、启用/禁用、删除自定义角色；
- 内置/自定义角色、作用域、绑定用户数、状态展示；
- 六页签编辑：基础信息、菜单权限、操作权限、数据范围、文件目录权限、绑定用户；
- 菜单、按钮、路由、API 权限配置；
- 数据范围与操作级别配置；
- `self`、团队/单位共享目录、团队/单位成员目录和全局文件策略；
- 绑定 admin/LDAP 用户及作用域；
- 权限矩阵同时显示菜单状态、最大数据范围和操作级别。

保存时依次调用角色基础信息、权限、数据范围、文件策略和用户绑定接口。任何一步失败均在界面明确报错，不伪造成功状态。

## 5. 文件管理器回归

- 路径和上一级按钮继续只消费后端返回的
  `effectivePath`、`initialPath`、`canGoParent`、`parentPath`。
- 上传、下载、新建目录、删除、复制、移动、重命名、打包下载、显示隐藏文件均增加 `data-permission`。
- 动态生成的文件行操作在渲染后重新应用按钮权限。
- 前端没有增加任何自行计算父目录或扩展文件边界的逻辑。

## 6. 新增迁移

- `007_frontend_route_permissions.sql`
- `007_frontend_route_permissions.down.sql`

迁移补全管理页面路由权限点，并根据旧角色已有菜单权限同步授予对应路由权限，避免双读切换后出现“菜单可见但路由被拒绝”。本批次未在现有业务数据库执行。

## 8. 风险与批次五事项

1. 007 up/down 迁移尚未在 PostgreSQL 隔离库实际执行和回滚；应纳入批次五迁移演练。
2. 当前仍为 legacy/shadow，前端 403 是体验层控制，接口最终拒绝仍由后端负责；切换 enforce 前需完整联调。
3. 角色“保存全部配置”由多个接口顺序提交，当前不是单个数据库事务。批次五应验证中途失败的恢复提示；后续可评估增加聚合事务接口。
4. 文件 TOCTOU 与 `openat/O_NOFOLLOW` 仍按批次三结论保留为 enforce 前安全评估项。
5. 团队/单位共享目录命名规则继续由后端文件策略解释器负责，前端仅配置策略，不硬编码实际有效路径。
