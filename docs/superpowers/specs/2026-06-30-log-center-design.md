# 日志中心设计

## 导航

新增一级菜单“日志中心”，位于“监控运维”和“系统”之间，二级菜单为：

- 用户登录日志
- 系统日志
- 审计日志

通过公共 `js/app.js` 动态注入，保证所有已登录页面一致；原“系统”菜单下的审计日志入口移除。

## 用户登录日志

PostgreSQL `auth_events` 持久化登录成功、登录失败和退出事件。字段包括账号、显示名称、账号类型、事件、结果、IP、User-Agent、会话标识摘要和时间；禁止保存密码与完整 token。仅管理员可查询，支持账号、事件、结果、时间和关键字筛选。

## 系统日志

后端只允许读取固定白名单来源：

- `simplehpc-backend`
- `slurmctld`
- `slurmd`
- PostgreSQL、Redis、OpenLDAP 容器

systemd 服务通过 `journalctl`，容器通过 `docker logs` 读取。接口限制时间范围、最大行数和输出长度，不接受任意服务名或命令。

## 审计日志

保留现有 `audit_logs`。新增统一 HTTP 写操作审计中间件，记录除认证接口以外的 POST、PUT、PATCH、DELETE 请求，包括操作者、路由、HTTP 状态、目标、IP 和时间；已有业务审计记录继续保留。

## 权限和界面

三类日志接口及页面仅管理员可用。页面统一支持刷新、筛选、分页、状态标签和详情预览；数据获取失败时明确显示错误，不回退到模拟数据。
